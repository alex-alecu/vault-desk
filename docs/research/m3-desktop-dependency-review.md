# M3 Desktop Dependency Review

Date: 2026-07-20

## Dependency Review

- Capability and milestone: M3 needs a small React frontend, deterministic web build, typed Tauri IPC, native folder/file dialogs, and the existing exact sidecar launch boundary.
- Existing repository alternative: the M0 Tauri smoke shell already pins Tauri 2.11.5, tauri-build 2.6.3, tauri-plugin-shell 2.3.5, and the Tauri CLI 2.11.4. Vitest already resolves Vite 8.1.4. There is no product React package or reusable desktop UI.
- Candidates and pins: React and React DOM 19.2.7; `@types/react` 19.2.17; `@types/react-dom` 19.2.3; `@tauri-apps/api` 2.11.1; Vite 8.1.4; `@vitejs/plugin-react` 6.0.3; esbuild 0.28.1; postject 1.0.0-alpha.6; tauri-plugin-dialog 2.7.2; and tauri-plugin-shell 2.3.5. Reuse the existing exact Tauri, build, CLI, esbuild, postject, and shell-plugin versions.
- Primary sources: [React versions](https://react.dev/versions), [React package](https://www.npmjs.com/package/react), [React DOM package](https://www.npmjs.com/package/react-dom), [Vite 8 release](https://vite.dev/blog/announcing-vite8), [official React plugin](https://vite.dev/plugins/), [Tauri JavaScript API package](https://www.npmjs.com/package/@tauri-apps/api), [Tauri dialog crate](https://docs.rs/crate/tauri-plugin-dialog/latest), and [Tauri capability permissions](https://v2.tauri.app/reference/acl/capability/).
- License and redistribution: React, React DOM, Vite, the React plugin, and TypeScript definitions are MIT. Tauri packages are MIT or Apache-2.0. All require normal notice/SBOM inclusion but no copyleft product obligation.
- Offline, telemetry, network, and credential behavior: the selected frontend libraries do not require runtime networking or credentials and add no telemetry. Vite, esbuild, postject, and Fast Refresh are build-only. No HTTP, updater, logging exporter, filesystem, opener, or JavaScript shell permission is added. The Rust dialog and shell plugins are called only by narrow Rust commands; the JavaScript bindings are deliberately not adopted because they would return raw paths or expose process launch to the webview.
- Footprint, native code, and platforms: React and the Tauri JavaScript API are small pure-JavaScript runtime dependencies. Vite, the official React plugin, esbuild, and postject are development-only; Vite 8 uses its existing native Rolldown toolchain. Tauri, dialog, and shell are native Rust dependencies supporting macOS and Windows. The product shell plugin launches only the exact packaged Core sidecar and is not granted to the webview.
- Security and maintenance: all candidates are official, actively maintained, and widely used. React 19.2.7 is newer than the patched 19.2.1 release for the 2025 React Server Components issue; this client-only app uses no server components. Tauri permissions remain deny-by-default and the product capability file grants only named custom commands. Raw selected paths remain in Rust/Vault Core.
- Adapter fit: React stays entirely in `packages/desktop`; `@tauri-apps/api` is wrapped by one `api.ts`; the Rust host owns dialog and sidecar APIs; Vault Core remains the product authority. No frontend dependency crosses into Core or workers.
- Research-derived claims to validate: WebView behavior, dialog UX, packaged sidecar lifecycle, bundle size, accessibility, focus restoration, and signing on physical macOS and Windows.
- Decision: adopt the exact pins above. Reuse existing Tauri, Vite, esbuild, postject, and shell-plugin resolutions. Register the shell plugin only with the implemented exact Core sidecar supervisor; keep shell permissions out of the webview capability. Reject the JavaScript dialog and shell bindings, routers, state libraries, component kits, CSS frameworks, filesystem plugins, HTTP plugins, updaters, analytics, and crash reporters for M3.
